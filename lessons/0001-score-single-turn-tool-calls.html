Lesson 0001

Score Single-Turn Tool Calls

Your next eval skill is to look at a model's selected tools and turn that behavior into a number. This is the bridge from "it seemed right" to an eval score you can compare across runs.

Reference: evals map Open evaluator code Open file tools data Primary transcript

The One Idea

Single-turn tool-selection evals can be deterministic because tool calls are structured objects. You are not scoring prose yet. You are scoring arrays like ["readFile"] against answer keys like expectedTools: ["readFile"].

The course transcript calls evaluators "the thing that turns the qualitative thing to a quantitative thing." In this repo, that means a function shaped like (output, target) => number.

Three Moving Pieces

Dataset The exam sheet. data is the question, target is the answer key.
Executor The runner. It shows data to the model and returns normalized output.
Evaluator The scorer. It compares output with target and returns 0 to 1.

Now Flip It

For negative examples, the target says which tools are forbidden. If the model selects any forbidden tool, the score should be zero. 

output.toolNames = ["listFiles"];
target.forbiddenTools = ["readFile", "writeFile", "listFiles", "deleteFile"];
target.category = "negative";

Your Next Code Move

In evals/evaluators.ts, write one small evaluator before touching Laminar wiring: 

export function toolsAvoided(
  output: SingleTurnResult,
  target: EvalTarget,
): number {
  // Return 0 if any forbidden tool was selected.
  // Return 1 otherwise.
}

Do not look for the model call here. This function should only inspect output.toolNames and target.forbiddenTools.
