The Pipeline

Piece Shape in this repo Job
Dataset entry { data, target, metadata } One exam question plus its answer key.
Data EvalData What the model sees: prompt, available tools, optional config.
Target EvalTarget What the evaluator sees: expected tools, forbidden tools, category.
Executor singleTurnExecutor(data) Runs one model call and returns normalized tool-call output.
Evaluator (output, target) => number Turns qualitative behavior into a quantitative score from 0 to 1.

Category Rules

Golden Strict. A clear prompt should select exactly the intended tool behavior.
Secondary Flexible. Useful to measure, but usually not punished as harshly.
Negative Guardrail. The agent should avoid tools that are outside the task.

Scorer Cheatsheet

Scorer Use when Return rule
toolsSelected You need a strict pass/fail check for expected tools. 1 if all expected tools were selected, else 0.
toolsAvoided You need a strict negative check. 0 if any forbidden tool was selected, else 1.
toolSelectionScore You want partial credit for secondary or ambiguous prompts. Use precision and recall to return an F1-like score.
